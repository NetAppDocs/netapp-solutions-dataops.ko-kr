== NetApp Cloud Manager를 사용하는 CVO 및 커넥터에 대한 AWS 인증 요구 사항

AWX/Ansible Tower를 통해 Ansible 플레이북을 사용하여 CVO 및 커넥터의 자동 배포를 구성하려면 다음 정보가 필요합니다.



=== AWS에서 액세스/비밀 키 획득

. Cloud Manager에서 CVO와 Connector를 배포하려면 AWS 액세스/비밀 키가 필요합니다.  AWS 콘솔에서 IAM-->사용자-->사용자 이름-->보안 자격 증명-->액세스 키 만들기를 실행하여 키를 획득합니다.
. 액세스 키를 복사하여 안전하게 보관하고 Connector 및 CVO 배포에 사용하세요.



NOTE: 키를 분실한 경우 다른 액세스 키를 생성하고 분실한 액세스 키를 삭제할 수 있습니다.

image:access-keys.png["새로 고침 토큰"]



=== NetApp Cloud Central에서 새로 고침 토큰 획득

. 계정 자격 증명을 사용하여 클라우드 중앙 계정에 로그인하세요. https://services.cloud.netapp.com/refresh-token[]
. 새로 고침 토큰을 생성하여 배포를 위해 저장합니다.


image:token-authentication.png["새로 고침 토큰"]



=== 클라이언트 ID 획득

. API 페이지에 접속하여 클라이언트 ID를 복사하세요. https://services.cloud.netapp.com/developer-hub[] .
. 오른쪽 상단에 있는 "인증 방법 알아보기"를 클릭하세요.
. 로그인에 사용자 이름/비밀번호가 필요한 경우, 팝업되는 인증 창에서 일반 액세스의 클라이언트 ID를 복사합니다.  SSO를 사용하는 연합 사용자는 "새로 고침 토큰 탭"에서 클라이언트 ID를 복사해야 합니다.


image:client-id.png["클라이언트 ID"]



=== AWS에서 키 페어 획득

. AWS 콘솔에서 "키 페어"를 검색하고 "pem"으로 키 페어를 생성합니다.  키 쌍의 이름을 기억해 두세요. 커넥터를 배포하는 데 사용됩니다.


image:key-pair.png["키 페어"]



=== 계정 ID 획득

. 클라우드 관리자에서 계정 –> 계정 관리를 클릭한 다음 AWX 변수에 사용할 계정 ID를 복사합니다.


image:account-id.png["클라이언트 ID"]
